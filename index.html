Circular Array Using Modulus

If we want our array to be circular. i.e when our last index is occupied and we want to insert any other value. we can add it the very start of the array (index 0),

and our second last element to the second index. etc. etc.

in this case we can use the Modulus operator, E.G if our array is of size 5 and we need to add a 6th elemnt. 6 mod 5 = 1, so index will equal 1 - 1 = 0

for the 7th elemnt. 7 mod 5 = 2, so index will equal 2 - 1 = 1


not only that we can also know what number of rotations an upcoming element is going to occupy using ceil(elementIdx/arraySize)

Please Note that elementIdx is not zero based.

E.G For an array of size 5 the 26th element is going to be inserted in the 6th cycle, i.e this is our 6th time we wraped around the array in a circular manner

Numerical Systems

How do bases work?

no matter what base system you are using. you are always limited by the Available number of symbols in that base system

E.G base 10 you have 10 symbols to work with. So as you are counting and run out of symbols, you simply add a new 'slot' and start counting from scratch. The leftmost slot is the most significant and holds the most weight in our number.

A standardized way to represent a Number is to use the base raised to progressive powers and adding them together

Kind of like representing currency with different Banknotes values.

E.G: 17412 in base10 is represented as 10^4*1 + 10^3*7 + 10^2*4 + 10^1*1 + 10^0*2

As you can see we have progressive 'tiers' of numbers (units, tens, hundreds, thosands.etc..) repeated to various degrees

Notice that: the various 'degrees/repititions' that our 'tiers' are repeated with are limited to our symbols

To represent 21 in base2 we can apply a similar Principle

our base is 2 so our 'tiers' will be powers of 2, we only have two symbols to work with (0, 1)

2^4*1 + 2^3*0 + 2^2*1 + 2^1*0 + 2^0*1

1 0 1 0 1


You can convert decimal to binary by asking yourself. what's the most significant tier of 2 that can fit inside my number

in our case with 21 the highest power of 2 that can fit inside 21 is 4. 2^4=16

then we make our way down the powers. if that power added to the previous power doesnt fit, we ignore it and move to a smaller power


another way to convert from base10 to a smaller base. is to divide our base10 number by the base we want to convert to. we keep note of the remainder each time. untill our qutioent is 0 then we stop

the remainders are the 'repititions' of the 'tiers'


For example:

The remainders from bottom to the top. are the binary representation of 21.

10101


Now let's apply this knowledge to a problem

Codeforces 2020A:Find Minimum Operations.


we are tasked to find the minimum number of times we can subtract k^x from n. such that n will equal 0. x is any positive integer.

Recall from the explanation above. that when representing a number in a different base k. we asked ourselves "what's the most significant power of k that can fit inside my number?" this is pretty much the solution of our problem, by representing n as a number in the base-K, we will go from the greatest powers of k that can fit inside our number and gradually go down. also recall that the digits in our slots represent the repititions of a specific power. the digits in the new slots are the remainders in the division method. so we will simply count the remainders as we are converting to base-K. these remainders are our number of steps


E.G: 1

Here the sum of remainders is 1. which indeed is the number of steps needed.

we can confirm this because the heighest power of 4 that fits into 16 is 2, so 4^2 = 16, only one step is needed

E.G: 2

what's the biggest power of 2 that can fit into 5? 2^2 = 4

after that, what's the next power of 2 that's when added to the previous power fits into 5? 2^0 = 1

our Modulus sum is 2, and indeed we only need 2 steps
